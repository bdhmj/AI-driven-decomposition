# Бэклог изменений — TG Estimator Bot

## 2026-03-26
- Создан проект: Telegram-бот для первичной оценки клиентских запросов на разработку
- Интеграция с Claude API (Sonnet) для анализа запросов, генерации ТЗ и декомпозиции
- Реализован полный флоу: /new_request → вопросы к клиенту → ТЗ → декомпозиция → корректировка ставок → маржа → итоговый отчёт
- ТЗ выгружается в формате .docx вместо текстового сообщения
- Переключение всех запросов к Claude API на модель Opus 4
- Добавлена кнопка "Ответов нет, далее" под вопросами к клиенту
- Кнопка "Заново" теперь запрашивает текстовые правки перед пересозданием ТЗ
- Итоговая таблица оценки в формате .xlsx по дизайну шаблона (шрифт Montserrat, оранжевые заголовки, чередование строк)
- Логотип MetaLamp добавлен в таблицу оценки (левый верхний угол)
- Деплой на сервер 188.116.21.247 (CentOS 8, /opt/tg-estimator-bot/, systemd-сервис)

## 2026-04-09
- Полный рефакторинг системы оценки: переход на внутренние ставки (на руки) вместо рыночных
- Новый шаг «Коэффициенты проекта» — PM настраивает 7 параметров (отладка, код ревью, коммуникации, QA, риски, DevOps, менеджер), из которых считается множитель K
- Новый шаг «Внутренние ставки» — настройка $/ч на руки для каждого специалиста на проекте
- Маржа агентства теперь применяется поверх внутренних ставок (клиентская ставка = внутренняя × (1 + маржа%))
- PM и QA рассчитываются автоматически по коэффициентам (PM — % от макс. специалиста, QA — % от общего)
- Декомпозиция теперь в днях (min/max), не в часах; задачи группируются по модулям
- Добавлен few-shot пример реального проекта (P2P Platform with OTC) в промпт декомпозиции для калибровки оценок
- Отчёт .xlsx: задачи группируются по модулям (вместо по специалистам), колонка «Оценка, дни» вместо часов
- Миграция сервера: 188.116.21.247 → 62.197.49.192

## 2026-03-30
- Поддержка файлов на входе: .txt, .docx, .pdf, .rtf, .odt, .md, .csv
- Поддержка Google Docs: бот извлекает текст по ссылке через export API
- Поддержка ссылок на сайты: бот обходит до 30 страниц (sitemap + внутренние ссылки), парсит контент, передаёт в Claude для формирования описания проекта
- Команда /skip — прерывает анализ на любой стадии
- Поддержка английского языка на входе: запрос может быть на любом языке, все результаты (ТЗ, вопросы, декомпозиция) выдаются на русском
